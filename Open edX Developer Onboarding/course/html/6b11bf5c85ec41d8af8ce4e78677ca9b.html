First, let’s come up with our definition of developer-friendly. Here’s what Open edX developers typically like to do day-to-day with an Open edX environment:

      • When we’re implementing a change for a service, we want to modify its source code with our preferred Integrated Development Environment (IDE) easily.
      • We want our development environment to hot-reload after modifying it.
      • In case we find a bug in our code, we want to debug it with our favorite debugging tool.
      • We want to install extra pip requirements, in editable mode if needed.
      • We want to modify the provision of the docker services if needed, so we can run services however we need to.

How do we do those things with Tutor? The Tutor CLI allows us to interact with our environment in a friendly manner! So we can do any of those things without far-fetched tricks. Let’s see how!

The Open edX services are hosted locally as Docker containers orchestrated by Docker compose, so the source code is embedded in the D images while building them.

As we usually do with Docker services, with Tutor we can use bind-mounts to connect the code in our machines to that provisioned in the Docker container. This way, we can change the service's source code or even the code from an installed library! This is how you do it:

First, copy the repository from the built image into your machine. We do this so we take all the necessary files to continue serving those services as usual:

tutor dev copyfrom --help
Usage: tutor dev copyfrom [OPTIONS] SERVICE CONTAINER_PATH HOST_PATH
Copy files/folders from a container directory to the local filesystem.
Options:
-h, --help Show this message and exit.
