To Learn More:

The Open edX project has over 200 repositories, and this course can’t possibly review all of them. This module highlighted the most frequently used ones, however, to learn more about the Open edX system, these additional repos are a good starting point:

    • XBlock Source Code: Reviewing the XBlock Source Code will make it easier to understand how XBlocks function; that knowledge will come in handy when debugging XBlock issues, either with your own blocks or others that may be installed on your instance.
    • Hooks extension framework: Hooks provide a stable set of places in the code where functions defined via plugins can execute. For a list of hooks used in the Open edX system, check out openedx-events and openedx-filters. The initial implementation of the Hooks framework was driven by eduNEXT.
    • New mobile apps: Recently the team at RaccoonGang has modernized our mobile applications and the Mobile App Working Group has additional enhancements planned. The current codebases are found in these GitHub: iOS mobile app code, Android mobile app code
    • New data analytics tools: A cross section of teams at Axim, eduNEXT, Overhang.io, and OpenCraft have been developing Aspects, an optional implementation of analytics for the Open edX LMS to replace Insights, which has been deprecated.
    • The future of the Open edX platform? A team of OpenCraft and Axim engineers have been developing openedx-learning, an effort to create a more nimble core learning platform, enabling rapid experimentation and drastic changes to the learner experience that are difficult to implement with Open edX today. This “modular learning” project is not ready for prime time; it is included here as a point of interest for those curious about where we might be heading over the next few years.
