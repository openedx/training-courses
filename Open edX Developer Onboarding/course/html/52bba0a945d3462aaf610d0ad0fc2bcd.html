Now that you have your Open edX instance up and running, you may want to make it look a little more like your organization’s branding. The Open edX system accomplishes this through theming. When theming an Open edX installation, you usually need to implement two distinct theming solutions:

      1. A “comprehensive theme” for parts of the system that are implemented as Django views
      2. A branding npm package for parts of the system implemented as MFEs.

Tutor supports both theming engines. To install a custom comprehensive theme, you need to make the source available to Tutor, and then install the theme with the Tutor settheme command as explained in the Tutor documentation.

To install a custom MFE branding package, you can utilize the patches provided by the Tutor-mfe plugin. See the documentation for more info.

Note: As of May 2024, a project has kicked off to utilize Design Tokens throughout our front end. This may change how theming works; please consult Tutor documentation and the Open edX forums for the most up-to-date information.
