Now that you know how to create a fork, let’s review a few other Git commands that you are likely to use: clone and  remote.

1. git clone

The git clone command creates a local copy of a remote Git repository. It allows you to obtain the entire repository, including all its files, commit history, and branches, on your local machine. To locally clone the fork you made in the previous step, use this command:

$ git clone https://github.com/USER/edx-platform.git

This command creates a new directory named ``edx-platform`` in your current location and clones the contents of the remote repository into it.
