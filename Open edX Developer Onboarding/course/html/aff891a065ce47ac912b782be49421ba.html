Developer Documentation

You should bookmark docs.openedx.org, our main documentation site for developers (and other personas, too). You’ll find a few different types of documentation on this site, described in more depth here:

      • Quickstarts: step-by-step instructions designed to help you learn a new concept or domain.
      • How-tos: guides that help you solve a specific problem.
      • References: information guides oriented to people who already know how to solve a problem, but need a refresher.
      • Explanation/Concepts: Teaching documents, such as this one on the Platform Overview All our docs are open source and we welcome contributions. If you see something wrong or missing, please open up a pull request and the Open edX Community will quickly take a look! You can use the GitHub button on the top of each page to open the source file in GitHub and suggest changes.

When you’re diving into code, take a minute to read each repo’s README file, rendered at the bottom of the repo homepage (here’s an example). The READ ME will explain what the repo is for and information like how to develop, how to contribute, what types of toggles/settings there are, and more.
