After installing your preferred Tutor version, to provision the Open edX services, run:

tutor dev launch

Running this command for the first time will do a couple of things:

    1. Generate a default config.yml which will be used to generate our environment
    2. Generate an env/ directory filled with the necessary templates to serve your Open edX installations. Tutor generates the environment by default in the default project root: ~/.local/share/tutor. This is how the directory looks like:

In the above directory:

            • The apps directory manages the overall configurations of the installation, in any mode, local, dev or k8s – short for kubernetes.
            • The build directory manages how to build images for the Open edX services.
            • The dev|local directories have the necessary docker compose definitions to deploy those environments.
            • The k8s directory has the necessary manifests to deploy a kubernetes environment.
            • The plugins directory has the necessary definitions (configurations, docker files, etc) for tutor plugins deployment.

3. Build the openedx, openedx-dev and other required images, set default configurations and start and initialize services. For a more detailed step-by-step description of what happens when you run tutor dev launch for the first time, see Tutor documentation: first-time setup.
