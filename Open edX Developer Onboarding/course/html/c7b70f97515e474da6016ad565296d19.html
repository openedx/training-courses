Then, mount what you copied into the services you want to modify:

tutor mounts add [OPTIONS] mount
tutor images build openedx-dev
tutor dev launch

After running those commands, we can modify the service's source code directly from your machine. Remember! This is recommended only for development environments, not production (live) environments.

Now, how can we implement the hot reload? Luckily, at least for the Django services, the command used to run it uses hot-reload out of the box. So, we can change the code, and it will reload by itself. If we want to do it manually, you can run:

tutor dev restart [OPTIONS] service

This is a more concrete example of the whole development environment setup process using the LMS service, try it out!

tutor dev copyfrom lms /openedx/edx-platform .
tutor mounts add ./edx-platform
tutor mounts list
