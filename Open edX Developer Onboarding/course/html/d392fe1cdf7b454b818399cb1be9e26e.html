When working with your fork as your remote for a repo, you will want to ensure you are still working with the latest code from upstream (the ``openedx`` version). You can do this within the GitHub UI. First navigate to your fork:

In the far right corner of the box that alerts you to how many commits your branch is behind the upstream, click the “Sync fork” button:

Success!

You can also accomplish the same task using command line instructions:

# Clone your fork and add the openedx remote
$ git clone git@github.com:YOUR_GITHUB_USERNAME/repository.git
$ git remote add upstream https://github.com/openedx/repository.git
# Fetch the latest changes from upstream
$ git fetch upstream
# Check out your fork’s local default branch, which may be “main” or “master” in the openedx repos
$ git checkout main
# Merge the upstream changes into your branch. Note: you should never put commits
# on your local main or master branch; you should always develop on a new branch with a unique name.
$ git merge upstream/main
# Push your updated (local) fork back up to your remote fork git push origin main

Note: If you are certain you have developed no code on the master or main branch, you can simplify these commands:

# If you know that you local repo didn't change
git checkout main
git pull upstream main
git push origin main
