This section reviews some common customizations that go beyond flipping feature toggles. To install a custom XBlock (or any other custom python package) to the LMS virtual environment, you need to add it to the OPENEDX_EXTRA_PIP_REQUIREMENTS configuration variable and redeploy the instance. Let’s say we want to install the v1.12.0 tag of xblock-poll. First open up your Tutor config file (here, we use the vi command line editor):

vi “$(tutor config printroot)/config.yml”

If the OPENEDX_EXTRA_PIP_REQUIREMENTS variable isn't already present, add it, and then add the requirement to it:

...
OPENEDX_EXTRA_PIP_REQUIREMENTS:
-
“git+https://github.com/open-craft/xblock-poll@v1.13.0”
...

You then need to rebuild the openedx image and restart Tutor:

tutor images build openedx
tutor local restart

Finally, to make the XBlock available in a course on your instance, you need to add it to the “Advanced Module List” setting in your course’s “Advanced Settings” page in the Studio. You need to do this for every course where you want the XBlock to be available.
