Developing plugins with Tutor

Tutor plugins are the officially recommended way of customizing the behaviour of Tutor. If Tutor does not do things the way you want, then your first reaction should not be to fork Tutor, but instead to figure out whether you can install or create a plugin that will allow you to achieve what you want.

You may be thinking that installing or creating a plugin might be overkill for your use case. It’s almost certainly not! The stable plugin API guarantees that your changes will keep working even after you upgrade from one major release to the next, with little to no extra work. Also, it allows you to distribute your changes to other users.

There are two ways to go about making plugins. For quick personal plugins, you can add Python modules directly to your local plugin root, as described here:

      • Plugin Tutorial (Tutor documentation)

For more complex plugins that you may wish to share with others, the Plugin Coookiecutter can help you get started:

      • cookiecutter-tutor-plugin (GitHub)

Of course, many plugins already exist, ready for you to install and use! Check out the official plugin guide for more details.
