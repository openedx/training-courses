Git Basics - Status

Now that you have accessed an Open edX repository where you want to write code, we need to talk about how we keep changes straight amongst so many different developers. For this, we utilize a Version Control System (VCS). There are a number of VCSs available like Mercurial, Git, etc. The Open edX project uses Git for our use case. 

In this unit, we'll learn some fundamental Git commands that will help you manage your code effectively. Before we dive into commands, let's understand the basic concepts of Git. Git is a version control system that tracks changes in your files and allows collaboration with others. It stores your code in repositories and keeps a history of changes.

There are several common Git commands you'll use regularly while executing the typical Git workflow. After making changes to your files, you need to stage them before committing. To see the status of your files, type:

$ git status

The git status command displays the current status of your repository. It shows which files are modified, which are staged, and which are untracked.  To see the changed lines of your files, type:

$ git diff

The git diff command displays the difference between your local, unstaged changes and what has been committed on your branch.

To stage changes, you can use the following command:

$ git add <filename>

By replacing <filename> with the name of the file (or using a period . to stage all changes), you add the changes to the staging area.
