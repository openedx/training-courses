Pre-requisites for a Kubernetes Deployment

When using Tutorâ€™s local deployment, docker-compose takes care of running and orchestrating the images that Tutor builds. When deploying to Kubernetes, running and orchestrating the images is the job of the Kubernetes platform.

In order to be able to use Tutor to deploy to Kubernetes, we need access to an available Kubernetes cluster. Popular providers that offer managed Kubernetes clusters are Amazon EKS, Google Kubernetes Engine (GKE), DigitalOcean DOKS, and Azure AKS.

When using Tutor to deploy to Kubernetes, Tutor assumes that you have kubectl installed in your path, and that kubectl is set up to have access to the cluster you want to deploy. The Tutor command basically delegates to kubectl when running k8s commands that interact with the Kubernetes cluster.
