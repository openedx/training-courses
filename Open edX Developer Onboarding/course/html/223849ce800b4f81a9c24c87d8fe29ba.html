MFE Toggles

Each MFE expects configuration that is independent from edx-platform. Right now, the easiest way to check what a particular MFE exposes is to go into each repository and look for a .env file. It will contain a list of environment variables that take strings as options. For example, see the Learning MFE's .env file, which contains variables that can be specified in your site, such as LOGIN_URL.

However, these files tend to be very sparsely commented. You can search the code in the repo for each variable to try and understand what they do, but in some cases, developers have done the right thing and documented them in the repository READMEs. The Course Authoring MFE README has some good examples; unfortunately, though, that is not the case for all repositories.

To complicate matters, some of these variables are actually used by sub-components, so searching the repository won't find what they're for. A good example of these would be the ones that frontend-component-header takes (LOGO_URL, SITE_NAME, etc).

As for the mechanism to actually set these, Tutor uses something called the mfe_config API, an edx-platform feature that lets operators set MFE environment variables (the ones described above) without having to edit the .env files directly and rebuild the MFE Docker image. This is documented in the corresponding ADR, but perhaps the best way to use it in practice is to follow what tutor-mfe suggests in its documentation. This is helpful for both site operators as well as developers looking to customize their local installation.
