In an open source project like the Open edX platform, documentation provides all community members with a common understanding of how each developer wrote their piece of the code and how all of the pieces of code work together. Good documentation provides clear directions and guidance to make any instance operate smoothly. The docs.openedx.org site contains main documentation on the Open edX system with different collections: for developers, documenters, educators, and system operators. Particularly, find the developer documentation here.

The Open edX Community is actively updating & migrating older documentation from a previous system into the docs.openedx.org site. Until that’s complete, you may find some of the read the docs documentation helpful (warning: it may be outdated). This older site was designed for course authors, but it may be helpful for you as you develop and test features.
