Backend Services

Testing provides important quality control on any changes you have made to the codebase, ensuring that everything functions as expected. For backend repositories, most repos use Makefiles to create commands to run tests. By and large, to run tests you’ll use make test in the repo root directory to execute tests.

In most repos, running make help from the root of the directory will show you what targets exist. However, the help directive is not implemented in all Makefiles, so inspecting the Makefile may be your only choice. (Note: we’d love a pull request that implements the help directive!)

If you need to, you can check out the repo’s Makefile (for example, xblock-lti-consumer's Makefile) to see what targets exist for running tests.

Running tests within the edx-platform repo works a bit differently. Within this repo you don’t use make commands, instead, you drop into a shell and run tests via pytest. See the Tutor docs on running edx-platform unit tests.
