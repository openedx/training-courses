This is it! The section you’ve been waiting for: Pull Requests!

If you’re relatively new to the developer world, you might not be familiar with this term. GitHub defines a Pull Request as “a proposal to merge a set of changes from one branch into another. In a pull request, collaborators can review and discuss the proposed set of changes before they integrate the changes into the main codebase.” If you haven’t opened a pull request before, check out Module 6: Opening A Pull Request, for more information on how to open a pull request to the upstream project from your fork. You may also want to review the “Contributing to the Open edX Project” section of the Open edX documentation site.

When you open a pull request, there are a few things you should do to make the process go as smoothly as possible:

1. Use draft pull requests when opening a pull request that isn’t ready to be reviewed yet. Sometimes you do this because you want feedback on in-progress work, or to run CI tests (next section). To convert a pull request to a draft, use the “Convert to Draft” button in the right hand sidebar of your PR:

2. When you open your pull request, write a good description! Many repos in the project have a pull request template, for example, the template for edx-platform. We humbly ask that you use the GitHub GUI to open pull requests, read the template for the appropriate repo carefully, and write as much detail as you possibly can. Screenshots and screen capture videos are great, when you can add them.

For more about submitting pull requests to an open-source project, this article from readwrite.com provides some great tips and tricks for getting your pull request description up to shape.
