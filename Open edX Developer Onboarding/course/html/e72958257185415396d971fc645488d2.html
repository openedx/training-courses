Tutor allows a developer to install any Open edX stable release, from Koa to the latest version (we encourage always using the latest version). It can also be used for development on top of the master branches, by using Tutor nightly. Therefore, when setting up an Open edX environment for the first time, you need to decide whether to use an existing release or the nightly branches of the Open edX project, this decision will determine the Tutor version to install to provision the Open edX services.

To help you decide, consider that existing releases are meant to be stable and production-ready, but they may not have the latest features. Conversely, while master has the latest changes, it may be unstable and contain bugs. Also, which option to use depends on your specific needs and goals. Here are a few use cases for each of them:

    • We want to use master when we are:
        • Contributing a new feature to the project
        • Contributing a bug fix or improvement
        • Testing out the latest features locally
    • We want to use an existing stable release when we are:
        • Backporting bug fixes to the latest stable releases
        • Developing new features, improvements or bug fixes for your initiatives

NOTE: As of December 2024, Tutor branches have been renamed from "nightly" and "master" to "main" and "release". See this forum post for more details. The following videos in this course were produced prior to December 2024, and may use outdated names for branches.
