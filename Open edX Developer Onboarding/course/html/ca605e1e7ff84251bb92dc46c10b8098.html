Using your own Python libraries in Tutor

What if I want to use another tool not built in – like pudb, as one of Python standard libraries? No worries, we can install it! This is what you can do:

        1. You can use your config.yml to modify your Open edX Dockerfile, by adding OPEN_EDX_EXTRA_PIP_REQUIREMENTS with a list of the requirements to install:

          OPENEDX_EXTRA_PIP_REQUIREMENTS:
          - pudb==2022.1.3
        2. Then run:
          tutor config save
        3. So you’ll get a new Dockerfile, which will install your requirements during the building process of the Open edX image. Now run:
          tutor images build openedx openedx-dev/span>
        4. Once you have your new image, you can check it has the requirements already installed by running:
          tutor dev exec lms pip show pudb

You can use pudb the way we used pbd, try it out! For more information about pudb, see its official documentation.

There’s a lot of stuff you can customize in your dev environment to make it more of your liking: Dockerfiles, docker compose files, Django settings, and even more! We’ll see some of those later on in the customization section.
