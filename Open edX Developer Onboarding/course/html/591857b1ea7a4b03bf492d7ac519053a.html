As with most open source codebases, the Open edX project operates on a CI/CD basis. Continuous Integration and Continuous Delivery (CI/CD) are terms used to describe a process where multiple changes are made to a codebase simultaneously. The Open edX project uses GitHub Actions to run our continuous integration (CI) testing. Additionally, as of 2024, the team at 2U/edx.org utilizes continuous deployment (CD) to deploy code as soon as it merges to the site. Their team is the only team in the Open edX ecosystem utilizing CD from master; most community members run off of Named Releases or their own forks.

      • Node.js CI tests runs Node tests on repos with frontend code
      • Python CI tests runs Python unit tests and pep8/pylint quality checks on Python based repos
