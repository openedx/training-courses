Several companies that run Open edX instances at scale have open sourced their custom solutions for automating infrastructure and/or extending Tutor’s capabilities for a more robust, production-ready deployment.

      • Grove: Grove is a tool that utilizes Terraform, Gitlab pipelines, and Tutor plugins to manage a Kubernetes cluster and managed services for databases, object storage, with built in backups and monitoring. It can manage multiple instances in a Kubernetes cluster and supports deployments to AWS and DigitalOcean.
      • Cookiecutter Open edX Devops Tools: This is a production-ready template that utilizes Terraform and Github actions to perform k8s-based installations of Open edX in AWS. The scripts are fully parameterized and can help you set up monitoring and autoscaling, among other features.
      • Drydock: Drydock contains a set of Tutor plugins that provide an option to manage deployment of instances through ArgoCD as a replacement for Tutor job commands. It also contains plugins that set up backups and monitoring, and comes with multi-tenancy support.
