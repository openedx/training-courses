Setting up the DNS entries for Kubernetes

Once you have access to your Kubernetes cluster, set up via kubectl, you can start using the tutor k8s family of commands. The first thing that you need to do is to get the IP of the load balancer which is the entry point into your cluster.

If your Kubernetes cluster doesn’t have an existing load balancer, Tutor can help you set up a caddy instance, which acts as a load balancer service and is also responsible for obtaining SSL certificates for the (sub)domains that your Open edX instance requires. To set up a caddy load balancer with Tutor run:

tutor k8s start caddy

Once the caddy instance is up and running, you can obtain its IP address by running:

kubectl --namespace openedx get services/caddy

Once you get the IP, you need to set an A record that points your main (LMS) domain to it, and a wildcard CNAME record that maps all subdomains to the main domain, just like we did in the local deployment example.
