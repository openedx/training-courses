The Open edX project started with a monolithic repository called edx-platform, which is still the largest and most actively developed repo in the project. However, we’re gradually moving away from this monolith and we encourage everyone to develop as much as possible in repos that aren’t edx-platform. A few alternative repos include:

      • MFEs: MFEs, or Micro Front-ends, encapsulate frontend functionality for many features that you see when using the Open edX platform. Lightweight and agile, MFEs enable faster development and deployment. The most visible MFE is frontend-app-learning, which powers the courseware pages you see (like the one you are reading right now!).
          • There are over twenty MFEs and counting! Review the current list to get a better sense of which Open edX functionalities are deployed via MFEs.
      • Paragon: An accessible, theme-ready design system built for the Open edX project specifically, Paragon’s reusable components provide a visually consistent user experience in MFEs. If you intend to code custom MFEs, Paragon will be integral to your work.
      • XBlocks: The building blocks of the Open edX courseware, XBlocks power everything that students interact with in the Open edX courseware. You’re currently looking at an HTML XBlock that has been configured to display text. So far in this course, you’ve also encountered an HTML XBlock configured as an iframe, the Video XBlock, the Problem XBlock, and the PDF XBlock. Browse the list of Open edX official XBlocks. Note that many other XBlocks have been and are currently being developed by the community.
