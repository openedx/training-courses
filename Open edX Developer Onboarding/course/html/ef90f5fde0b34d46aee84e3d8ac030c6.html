Accessibility (a11y)

Coding with accessibility (or “a11y”: “a” + 11 letters + “y”) in mind means ensuring that the features you build can be used by people accessing the site in various ways, for example via use of assistive technology like screen readers. Accessibility primarily impacts how we build frontend features, although that is not always the case.

Some resources we have around a11y:

      • Open edX Accessibility Concepts 
      • Open edX Accessibility Reference

The Paragon pattern library defines various elements (such as buttons and text boxes) that we use throughout the frontend; these elements are guaranteed to have a11y built right in. When designing for the frontend, if you use paragon components you're less likely to run into a11y issues than if you roll your own. If you have questions, we recommend you get in touch with the Frontend Working Group.
