Here is a great usage of comments:

# Cache a new event id and event type which the signal handler will use to emit a tracking log event.
create_new_event_transaction_id()
set_event_transaction_type(events.SUBSECTION_OVERRIDE_EVENT_TYPE)

This comment helpfully explains why two lines of code need to be run. It may not be obvious when just seeing these two lines, but because having them there is critically important, we leave a comment to make sure everyone understands why they’re there.
