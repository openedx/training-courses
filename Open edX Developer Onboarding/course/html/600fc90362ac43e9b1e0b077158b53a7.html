Once the domains and subdomains have been set up, you are ready to install the other system requirements. Tutor requires Docker Engine, libyaml development headers, and Python. Check Tutor docs for details about which versions of Docker and Python are supported.

We assume that you have access to a fresh VM with Ubuntu Server (version 22.04 (LTS) or newer LTS) installed, ports 80 and 443 accessible from the internet, and DNS records set up.

If you are using a different operating system, you might have to adapt the steps below to work for your distribution. The Docker documentation contains installation instructions for a vast list of different operating systems. The method of installation and package names of Python 3, PIP, and libyaml dependencies can also be different if you’re using a different Linux distribution.

The first step is to install Docker Engine following the instructions “Install using the apt repository” from Docker documentation.

Next we install python3, pip, and libyaml development headers:

sudo apt-get install python3 python3-pip libyaml-dev

In order to be able to run docker commands, the Linux user that you will use to run Tutor, needs to be added to the docker user group:

sudo usermod -aG docker $USER

At this point, you should log out of your account, and log back in to make sure that your user environment gets properly reset.

Your VM is now fully prepared for Tutor.
