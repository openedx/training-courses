Now that we have our installation ready to use, what can we do? This section explores the commands we typically use and the documentation for each one!

1. Visit the provisioned services

Now that you have provisioned your services, you should be able to visit each service on your localhost. After provisioning the services by running tutor dev launch, you’ll get an output like this, which will show you the domains of each service that is running:

All services initialised.
The platform is now running and can be accessed at the following urls:
http://local.overhang.io:8000 http://studio.local.overhang.io:8001 http://apps.local.overhang.io:1999/authn http://apps.local.overhang.io:1997/account http://apps.local.overhang.io:2001/course-authoring http://apps.local.overhang.io:2002/discussions http://apps.local.overhang.io:1994/gradebook http://apps.local.overhang.io:2000/learning http://apps.local.overhang.io:1995/profile http://notes.local.overhang.io:8000

Once you’ve provisioned, you can view the domains for the services provisioned at any time by checking your config.yml for the services’ domains, or by using the command tutor config printvalue <SERVICENAME_HOST>:

tutor config printvalue LMS_HOST
tutor config printvalue CMS_HOST

Review the documentation for the tutor config commands.
