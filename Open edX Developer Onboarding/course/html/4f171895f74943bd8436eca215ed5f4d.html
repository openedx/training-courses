Since we can modify the service source code and previously installed libraries in Tutor, we can have a more straightforward debugging process. Imagine we found a bug in the registration page, and we figure it’s related to the backend used by the page. How can we check what’s happening? Try the following:

  1. First, try exploring the Open edX project’s repositories. Specifically, the edx-platform repository, so you get familiar with the project’s structure. That repository implements the main Open edX services: the Learning Management System (LMS) and the Content Management Service (CMS) that powers Open edX Studio. You’ll find that both are a sum of multiple Django applications, some of them specific to each service and some shared between them.
  2. Once we find the Django view we want to debug, in our case class RegistrationView (API View) (don’t worry about finding the right view yet, it will get easier once you get familiar with the project), add a breakpoint at the start of the view (the first line within the post method). This way, we can check line by line what’s happening inside the view:
breakpoint()

3. Now, you need to attach a terminal to the standard output of the service you want to debug. Since we’re debugging the LMS service, do this:

tutor dev logs lms -f
