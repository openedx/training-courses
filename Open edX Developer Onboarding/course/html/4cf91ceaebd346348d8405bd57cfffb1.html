To Learn More:

Once installed, you can start your Tutor development environment by running tutor dev start or tutor dev launch.

Using launch is good in a few scenarios:

      • You've changed something, such as your list of installed plugins, and you want to be sure that images are updated and init is run.
      • You haven't used tutor in a while and you want to bring everything up-to-date.
      • Something's wrong and you just want to refresh everything without having to delete your database.
In every other case, we recommend using start. For context, one Open edX developer reported using launch once a week, and start multiple times a day. Explained a different way, "start is faster, launch is more comprehensive".
