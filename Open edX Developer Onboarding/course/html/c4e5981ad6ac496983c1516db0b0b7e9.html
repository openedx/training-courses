Tutor strives for simplicity and ease of use, and for that reason does not come with all the advanced capabilities that you might want if youâ€™re setting up a large Open edX instance.

For production-grade deployments, you will want to set up backups, monitoring, and cron jobs. Depending on the volume of your instance, you might also want to configure k8s autoscaling.

Tutor does not include these capabilities, but it instead encourages creating such add-ons using the Tutor plugin architecture. A community supported set of add-ons for production environments is being developed under the Harmony Project (more on this later in this section).
