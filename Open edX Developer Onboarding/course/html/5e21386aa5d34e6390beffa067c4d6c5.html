Here is a workflow demonstrating git clone and git remote. In this example, we use the option depth=1 to make the clone step go faster. This only clones the master branch and is much faster, however, we recommend doing full clones for your environment.

Setting up a remote is very useful when you are working with a Fork of a repository where you need to pull the main branch of the main repository and you need to push the changes to the Fork. Start with cloning the repository. This forked repo exists at https://Github.com/farhaanbukhsh/edx-platform.

$ git clone https://github.com/farhaanbukhsh/edx-platform.git

This will clone the repository and will add a default remote called origin. Next, add the upstream repository, openedx/edx-platform, as the upstream:

$ git remote add upstream https://github.com/openedx/edx-platform.git

And it will look like:

Using this you can make changes to a branch and push it to your fork using:

$ git push origin <branch_name>
