2. Using the Tutor CLI to explore docker services

What if you want to interact with the services’ containers? You might want to execute a command in a running container:

tutor dev exec –help
Usage: tutor dev exec [OPTIONS] ARGS...
Run a command in a running container. This is a wrapper around `docker- compose exec`. Any option or argument passed to this command will be forwarded to docker-compose. Thus, you may use `-e` to manually define environment variables.
Options:
-h, --help Show this message and exit.
tutor dev exec lms bash
tutor dev exec lms ./manage.py lms shell

Or attach to a services’ logs:

tutor dev logs lms -f

And a lot more! Review the documentation for the tutor dev commands.
