Internationalization

For many organizations, the internationalization of the Open edX software has been a key benefit of adopting it as their LMS. The Open edX software is globally used and has already been translated into dozens of languages using the crowd-sourcing software Transifex. As of March 2024, 18 languages are officially supported by the Translations Working Group, including French and Spanish (Latin America).

See the up-to-date list of officially supported languages here.

It’s important to note that the only way that a site can be translated is if its strings are marked appropriately for translation. There is special syntax in Python/Django and JavaScript/React to define strings in a way that tells a scraping script to scrape the strings into a translation file. In simple terms, translators translate these strings into the desired language, the translated strings are returned to the translation files, and software serves strings according to user preference. The actual coding of the process is a bit more complicated than that; Module 6 > Internationalization reviews how it works in greater detail.

Note that the term Internationalization is often abbreviated to “i18n” - “i” followed by the number 18 (representing the 18 letters in the middle of the word) followed by “n”. You’ll see this abbreviation a lot.

When preparing strings for translation, pay close attention to interpolation, plural forms, and providing contextual comments for translators.
