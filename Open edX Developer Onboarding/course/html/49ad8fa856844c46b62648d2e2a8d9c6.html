Using Tutor’s local deployment for production is good enough for small instances with modest requirements and low number of users, but the simplicity of Tutor’s local deployment, which runs everything on the same server, comes with some disadvantages:

      • Scaling to a larger number of users is challenging.
      • Such a system is not resilient to hardware or network failures. A common engineering mantra is that a "cluster is not a cluster under three nodes".

If you need a more performant and resilient deployment, Tutor supports deploying to Kubernetes clusters.

Kubernetes is a powerful but complex system. This section assumes that you are already familiar with the basic concepts of Kubernetes. If you are not familiar with it, or need to refresh your memory, the official Kubernetes site contains a section explaining the basic concepts of Kubernetes and the Linux Foundation offers a free course on edx.org.
