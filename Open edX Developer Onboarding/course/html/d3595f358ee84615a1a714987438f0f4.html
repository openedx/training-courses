To Learn More:

  • A community member’s advice on learning to use Git:

“I always tend to recommend the official Git Book. … I personally struggled with Git initially, before one evening I decided to spend just a couple of hours reading the official book, and then followed it up with some practice at work (at that place we used to use SVN (Subversion), and I really hated that I couldn’t do branches easily, so when I learned you can use Git on top of the SVN and just push it to SVN, I was delighted and it made a huge difference for me). Since then, I felt pretty confident in using Git CLI (I tried GUI and TUI, and it just didn’t do it for me, except maybe for resolving huge merge conflicts)."

“For the basics, chapters 1, 2, 3 and 5 are a must. Everything else I think is optional. For the advanced, it really depends on your use case - most of the people I’ve talked to just do add, commit, push.”
