First thing we will do is set up DNS records for the domain and subdomains via which the instance is going to be accessible from the internet. Ensuring that the DNS is set up before launching Tutor makes it possible for Tutor to immediately request SSL certificates for your instance. Otherwise, you may have to wait for DNS cache invalidation.

A typical installation uses a main hostname for the LMS and subdomains of the main hostname for other services such as the Studio and various MFEs.

Let’s say your domain is “www.myinstance.com”. You will have to create an A record pointing myinstance.com to the public IP of your Open edX server and a wildcard CNAME record, pointing *.myinstance.com to the main hostname. At a minimum, you need to set up the following subdomains:

      • myinstance.com = LMS
      • *.myinstance.com = everything else (studio.myinstance.com, courses.myinstance.com, and many other subdomains than various Micro Front-ends (MFEs) utilize)

The Tutor settings are: LMS_HOST / CMS_HOST. The relevant information about TLS is here: SSL/TLS certificates for HTTPS access

The way to set up these DNS records depends on your DNS provider and is out of scope of this module. Search your DNS provider’s support site for clarification on the exact methods recommended.

This course does not address how to set up the marketing site and/or ecommerce functions on your domain that points at the LMS. Given the multiple, complicated ways that exist to approach these features, contacting a service provider or experienced Open edX developer may be your best option if you want a customized marketing site in place of the default Open edX course catalog page.
