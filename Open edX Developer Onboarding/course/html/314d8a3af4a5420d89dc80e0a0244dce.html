It’s common that you’ll need to run multiple versions of the Open edX platform - for example, you may need to run the most recent Named Release to simulate your installation, while also running nightly in order to develop features for the upstream Open edX codebase. Imagine we’re installing both Tutor versions (the most recent release and nightly) in two different virtual environments. We can’t use the default path for generating our Tutor environments, since they’ll override each other. So, we need to tell Tutor where we should generate our environment, among other things:

      1. We need to export the environment variable TUTOR_ROOT. So that configuration, environment, and data are not mixed up between platforms.
      2. Then, we must set up a DEV_PROJECT_NAME in our config.yml. So the various docker-compose projects don’t share the same name and you can have multiple environments that don’t get mixed up.

For more information about Tutor customization, please refer to Configuration and Customisation  and Running Multiple Open edX Platforms on a Single Server

Then, we can run tutor dev launch in each environment and get two different tutor installations. Keep in mind, however, that you can’t run the tutor dev launch command simultaneously since they use the same ports from your host machine unless you configure them otherwise.
