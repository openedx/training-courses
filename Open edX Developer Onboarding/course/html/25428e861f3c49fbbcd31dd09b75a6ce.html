Welcome to Module 3: Site Operators

This module focuses on people who are interested in running their own Open edX site. The Open edX community often calls these people “site operators” or “DevOps engineers”. This module is optional; if you are a developer but won’t actually be maintaining your own site, you can move on to Module 4.

By the end of this module, you should be able to recognize what it means to deploy an Open edX instance, demonstrate the basic steps to deploy an Open edX site, and identify where to look for additional help.

Note: this course often uses “Open edX instance” and “Open edX site” interchangeably, but in DevOps terms there is a difference. An "instance" refers to a single installation of the Open edX software. A single Open edX instance can be configured to run multiple Open edX "sites" (see Configuring Open edX Sites). Two sites can appear to be completely separate to the end user (using different domains and styling) while being powered by the same instance on the backend. One common use for this functionality is to provide a version of your content in a different language, another might be to allow targeted programs within your organization to feature their specific programming.
