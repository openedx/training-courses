Tutor is a Python-based library that makes interacting with the Open edX platform easy. It helps automate the deployment, management, and scaling of Open edX installations. Basically, Tutor takes care of three things:

1. Project configuration: User-specific settings are stored in a single file – called config.yml. This configuration file is all you need to generate your environment for development or production installations. Here, you’ll specify how your installation will behave, from secrets to host configurations.

# config.yml
CMS_HOST: <CMS HOST>
CMS_OAUTH2_SECRET: <OAUTH2 SECRET>
CONTACT_EMAIL: <CONTACT E-MAIL>
ID:<RESOURCES ID>
JWT_RSA_PRIVATE_KEY: '-----BEGIN RSA PRIVATE KEY----- -----

<RSA PRIVATE KEY TEXT>

-----END RSA PRIVATE KEY-----'
LANGUAGE_CODE: en
LMS_HOST:<LMS HOST>
MYSQL_ROOT_PASSWORD:<MYSQL PASSWORD FOR ROOT USERS>
OPENEDX_MYSQL_PASSWORD: <MYSQL PASSWORD>
OPENEDX_SECRET_KEY: <SECRET KEY>
PLATFORM_NAME:<YOUR PLATFORM NAME>
PLUGINS:
- mfe
PLUGIN_INDEXES:
- https://overhang.io/tutor/main

2. Template rendering: Tutor implements a set of templates that are used to generate your environment based on the user-specific settings you defined in your config.yml file.

3. Command-line interface: The library offers a CLI for managing Open edX installations. From deployment to scaling. You can perform various tasks using the CLI by running:

tutor [OPTIONS] COMMAND [ARGS]
