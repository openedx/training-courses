Opening A Pull Request from a Fork

There are a few different ways to open a pull request to an upstream repository from your fork (see Module 4, Forking Repositories, for more on forks). See also Quick Start: First Open edX Pull Request 

      1. Use the GitHub UI. This can be useful when you’re just updating documentation or making a very short fix. This tutorial will show you how to open a PR using the GitHub UI.
      2. A tutorial we have helps you open your first development PR.
      3. Or, fork the repo, locally clone it, do your work, and commit your changes to your branch, as discussed in Module 4. The next step is to visit your fork on GitHub and use the UI to open your pull request - you will see a yellow banner on the top of the page: 

Click the “Compare & pull request” button to open a PR. There are a few more steps; see this GitHub tutorial for all the steps.
