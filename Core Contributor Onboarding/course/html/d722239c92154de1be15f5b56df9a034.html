If you're reading this, congrats - you've been accepted as a Core Contributor to the Open edX project, fulfilling a specific role. The project has many types of CC roles; you can read about them in detail on the Core Contributor role definition page. To pick a few examples, we have roles that include:

  • Project managers lead specific Open edX projects and/or coordinate working groups and run their meetings
  • Product managers help run the development process for new features and major improvements. They help collaboratively build the roadmap for the project.
  • Code contributors (or programmers or developers) work on the source code of the Open edX project. They develop new features and fix bugs across the whole platform, from the whole of the courseware system down to individual XBlocks. They work on both things for upcoming releases as well as fixes to named releases.
  • Release Managers run the release process for the Build-Test-Release (BTR) group. This position enables them to fully create, modify, fix, and distribute the release as needed.
  • Content Authors create courses on this site! They are responsible for taking the time to ensure our training courses have accurate and up-to-date information presented in a clear and engaging manner.

We encourage you to check out the full list. Although you've currently been elected to one role, you are allowed to hold many roles if you are elected to each of them.
