Coding Standards

There are lots of coding and best practice standards that the Open edX project has, in order to keep code consistent, clean, and as free of bugs as possible. Many of these are broadly defined in OEPs (Open edX Enhancement Proposals) and ADRs (Architectural Decision Records). Some of these include:

  • OEP-10, Open edX Releases
  • OEP-13, API conventions
  • OEP-49, Django App Patterns
  • OEP-51, Conventional Commits

There are a lot of OEPs, and we don't expect you to read every single one. It's good to familiarize yourself with the site, and know to look to it as a resource when you're looking for major architectural guidance.
