In this module, we learned:

  • Introducing yourself on the forums and attending working group meetings are great ways to get started with the community
  • We've provided a handy Resource Reference for all contributors
  • For those of you interested in contributing code to the Open edX project, we've prepared a Developer Onboarding course to help you orient yourself with the coding community, our tech stack, our developer environment (called Tutor), and our coding conventions.
  • For coders, this Quickstart will walk you through your first Open edX pull request.
  • Working Groups are the best place to find things to work on. See the Working Groups section for more detail. If you can't make a synchronous Working Group meeting, post in their Slack room for information on where they could use help.
